> 실기기(iPhone, Xcode Release 빌드)에서 **ko / en 각각 1회씩** 완주.
> 각 단계 담당 코드는 감사 완료(전부 연결·테스트 존재). 아래는 실제 디바이스 육안 검증용.

- [ ] `supportsTablet: false` 반영 빌드인지 (iPhone 레이아웃, 아이패드 스케일 아님)
- [ ] 약관/개인정보 화면에 **최종 수정일 2026년 7월 24일 / July 24, 2026**, 운영자 **김정원 / Jeongwon Kim** 표시, `[ ]` 플레이스홀더 없음

## 핵심 경로 (순서대로)

### 1. 위치 권한 거부 → 수동 검색 폴백
- [ ] 코스 생성 진입 후 위치 요청에서 **거부** 선택
- [ ] 권한 안내 Alert 노출(문구 자연스러움), 앱 크래시 없음
- [ ] 수동 검색으로 계속 진행 가능

### 2. 수동 장소 검색
- [ ] 2글자 이상 입력 시 결과 표시, 거리(m/km) 표기
- [ ] 최근 검색 저장·재노출
- [ ] 결과 없음/네트워크 지연 상태 깨지지 않음

### 3. 추천/코스 생성
- [ ] 조건 입력 후 생성 → generating 진행바 정상, 결과 코스 2~4스텝
- [ ] 생성 실패(예: 기내모드) 시 오류 문구·재시도 안내 노출

### 4. 스텝 교체
- [ ] 스텝 액션 시트 → "교체" → 후보 시트 노출(2~3초 내)
- [ ] 후보 선택 시 해당 스텝만 교체 반영, 잔상 모달 없음

### 5. 스텝 잠금
- [ ] 스텝 잠금 토글 → 잠긴 스텝은 교체/삭제 비활성
- [ ] 재생성해도 잠긴 스텝 보존

### 6. 부분 재생성
- [ ] 특정 스텝만 재추천 → 잠긴 스텝 그대로, 대상만 변경
- [ ] 전체 재생성도 정상

### 7. 코스 확정
- [ ] 확정 전 액션에서 "먼저 확정" 가드 노출
- [ ] 확정 → 제목 입력/반영, 확정 상태 배너
- [ ] ⚠️ 확정 경로 2개(course-result 세션 confirm vs card/confirm 계획 confirm) 중 **출시 기준 흐름 확인** — 아래 미결 참고

### 8. 외부 리뷰 열기
- [ ] 장소 상세에서 카카오맵 링크 → 인앱 브라우저 정상
- [ ] 교체 후보의 네이버 리뷰 링크 정상

### 9. 피드백 / 반응 / 별점
- [ ] 카드 반응(하트 등) 토글·해제 정상, 낙관적 업데이트
- [ ] 리뷰 별점(1~5) 저장 → 카드 done 처리, "다시 가고 싶음" 파생

### 10. 회원 탈퇴
- [ ] 사유 선택 + 동의 체크 후 탈퇴
- [ ] 서버 삭제 성공, 로그아웃/초기 화면 복귀
- [ ] 실패 시 Alert(재시도 가능)
