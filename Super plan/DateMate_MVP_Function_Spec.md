DateMate MVP 기능명세서
커플 데이트 계획 부담 완화 앱 · MVP 범위 기준

MVP 목표: “커플 연결 후 7일 안에 둘 다 반응한 데이트 후보 3개 이상”이 만들어지는지 검증한다.

1. MVP 범위
구분	포함	제외
계정/연결	소셜 로그인, 닉네임, 초대 링크, 커플 연결	프로필 꾸미기, 공개 커뮤니티
온보딩	기본 취향, 피하고 싶은 조건, 장거리 여부	복잡한 심리 테스트
데이트 모드	앱이 골라줘, 느낌만 말할게, 코스로 정리해줘, 부드럽게 말해줘	고도화된 위치 자동 추천
AI	조건 해석, 카드 생성, 문장 생성, 추천 이유	SNS 트렌드 자동 수집
후보 관리	반응, 후보 분류, 둘 다 괜찮은 후보 모음	실시간 채팅, 상세 캘린더

2. 핵심 사용자 시나리오
시나리오	설명
계획 부담 완화	사용자가 “앱이 골라줘”를 선택하고 컨디션/예산/거리/시간만 입력하면 후보 3개가 생성된다.
수동적 참여 유도	사용자가 “느낌만 말할게”를 선택하고 분위기 버튼만 눌러도 데이트 카드가 생성된다.
의견 표현 도움	사용자가 “오늘은 조금 부담돼”를 선택하면 앱이 이유를 묻고 부드러운 문장을 생성한다.
아이디어 구체화	사용자가 “전시회 가고 싶어”처럼 러프하게 입력하면 예상 시간/비용/준비물이 포함된 카드로 변환된다.
장거리 커플	“다음에 만나면” 모드에서 다음 만남 버킷리스트와 원격 데이트 후보를 저장한다.

3. 기능 상세 명세
ID	기능	상세	우선순위
F-001	소셜 로그인	카카오/애플/구글 로그인. 최초 가입 시 닉네임 입력.	MVP 필수
F-002	커플 연결	초대 링크 또는 코드 생성. 상대가 가입/로그인 후 커플 공간 생성.	MVP 필수
F-003	기본 취향 온보딩	데이트 선호, 피하고 싶은 조건, 장거리 여부, 계획 성향 입력.	MVP 필수
F-004	데이트 모드 선택	오늘 필요한 도움을 선택. 모드별 입력 폼 다르게 노출.	MVP 필수
F-005	느낌 입력	자유 텍스트 또는 버튼으로 컨디션·예산·거리·분위기 입력.	MVP 필수
F-006	AI 조건 해석	입력값을 energy/budget/distance/preference/avoid JSON으로 변환.	MVP 필수
F-007	데이트 카드 생성	제목, 요약, 예상 시간/비용, 태그, 추천 이유 생성.	MVP 필수
F-008	부담 없는 반응	완전 끌려/느낌은 좋아/오늘은 부담돼/다음에 등 반응 저장.	MVP 필수
F-009	내 마음 문장 생성	피곤함·예산·거리 등 민감 조건을 부드러운 문장으로 생성.	MVP 필수
F-010	후보 분류	둘 다 끌림, 조건부, 다음에, 장소만 변경 등 자동 분류.	MVP 필수
F-011	추억 기록	완료한 데이트에 후기·다시 하고 싶은지 저장.	MVP 선택
F-012	외부 지도 연결	장소 검색은 네이버/구글맵 외부 링크로 연결.	MVP 선택

4. 데이터 모델 초안
엔티티	주요 필드	설명
User	id, nickname, auth_provider, created_at	개별 사용자
Couple	id, user_a_id, user_b_id, invite_code, status	커플 연결 단위
PreferenceProfile	user_id, preferred_tags, avoid_tags, budget_level, distance_sensitivity	개인 기본 취향
DateModeSession	id, couple_id, mode, input_json, created_by	모드별 입력 세션
DateCard	id, couple_id, title, summary, budget, duration, tags, status	AI 또는 사용자가 만든 데이트 후보
Reaction	id, card_id, user_id, reaction_type, note, created_at	상대 반응 및 조건부 선호
GeneratedMessage	id, user_id, card_id, input_reason, output_text, sent_at	내 마음 문장 생성 기록

5. AI 출력 스키마
조건 해석 JSON 예시
{
  "energy": "low",
  "budget": "low",
  "distance": "near",
  "preference": ["food", "comfortable"],
  "avoid": ["long_walk", "expensive"]
}
데이트 카드 JSON 예시
{
  "title": "동네 맛집 포장 + 집 영화",
  "summary": "멀리 가지 않고 맛있는 걸 먹으며 편하게 쉬는 데이트",
  "estimated_time": "2~3시간",
  "estimated_budget": "1인 1~2만 원",
  "tags": ["피곤한 날 가능", "이동 적음", "돈 적게 듦"],
  "why_recommended": "오늘은 오래 이동하기보다 가까운 곳이 잘 맞아 보여요."
}
6. 화면별 기능 매핑
화면	주요 기능	비고
홈	모드 진입, 오늘의 후보, 상대 반응, 둘 다 끌린 후보	첫 행동 유도
데이트 모드	모드 선택, 조건 입력, AI 후보 생성	핵심 입력 화면
우리 후보	후보 리스트, 반응 상태, 조건부 분류	합의 가능한 후보 정리
마음 전하기	민감 조건 선택, 문장 생성, 수정 후 전송	자동 전송 금지
추억	완료 기록, 후기, 추천 반영	MVP 선택 기능

7. 성공 지표
지표	목표/의미
커플 연결 완료율	초대한 상대가 실제 연결까지 완료하는 비율
첫 모드 선택률	사용자가 도움 선택 구조를 이해했는지 확인
첫 카드 생성률	핵심 행동 발생 여부
상대 반응률	둘이 함께 쓰는지 확인
내 마음 문장 사용률	의견 표현 기능의 실사용 여부
